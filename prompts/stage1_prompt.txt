## 4. Project Responsibilities

**AI.Factory.Api**

* Entry point API
* Endpoints:

  * POST /api/request → publishes `UserRequestReceived`
  * GET /api/tasks
  * GET /api/events
  * GET /api/agents

**AI.Factory.Core**

* Base domain models:

  * Agent
  * TaskItem
  * Module
  * Plan
  * ExecutionLog
* Interfaces:

  * IAgent
  * IEvent
  * IRepository

**AI.Factory.Events**

* Event contracts:

  * UserRequestReceived
  * RequirementAnalyzed
  * BusinessPlanCreated
  * TechnicalPlanCreated
  * TaskGraphBuilt
  * TaskReadyForExecution
  * TaskCompleted
  * TaskFailed
  * ReviewRequested
  * ReviewPassed
  * ReviewFailed

Each event includes:

* EventId
* Timestamp
* Source
* Payload

**AI.Factory.Agents**

* BaseAgent abstract class
* AgentRegistryService
* Scans `agents/specs/` for YAML definitions

**AI.Factory.Orchestrator**

* Placeholder for IntentRouterService & MasterOrchestratorService

**AI.Factory.TaskEngine**

* Models:

  * TaskItem
  * TaskDependency
  * TaskStatus (Pending, Ready, InProgress, Completed, Failed)
* Interface: ITaskGraphService

**AI.Factory.ModelManager**

* Base structures:

  * ModelProvider
  * ModelMetrics
  * ModelSelectionPolicy

**AI.Factory.ContextEngine**

* Placeholder for context/RAG engine
* Services:

  * ContextRetriever
  * ContextIndexService

**AI.Factory.Infrastructure**

* Database configuration
* Event bus configuration
* Logging configuration
* Repository implementations

## 8. Agent Specification System

* YAML specs in `agents/specs/`
* Example:

```yaml
agent_id: AGENT-BA
name: BusinessAnalystAgent
event_subscriptions:
  - UserRequestReceived
event_outputs:
  - BusinessPlanCreated
capabilities:
  - requirement_analysis
```

* Registry scans this folder dynamically.
